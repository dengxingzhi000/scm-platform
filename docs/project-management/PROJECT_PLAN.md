## 1. 项目概览

### 1.1 项目目标

**业务目标**:
- 构建对标阿里菜鸟、京东物流的企业级供应链管理平台
- 支持商品管理、库存管理、订单处理、仓库管理、物流跟踪

**性能目标**:
- 网关 QPS > 100,000
- 订单创建 TPS > 10,000
- 库存扣减 TPS > 20,000
- 商品搜索 QPS > 50,000
- P99 延迟 < 100ms
- 可用性 > 99.9%

### 1.2 项目范围

**包含在内 (In Scope)**:
- ✅ 商品服务（CRUD + Elasticsearch 搜索）
- ✅ 库存服务（原子扣减 + 预占释放）
- ✅ 订单服务（状态机流转 + 分布式事务）
- ✅ 仓库服务（入库/出库/调拨）
- ✅ 物流服务（轨迹查询 + 定时同步）
- ✅ 供应商服务（供应商管理 + 采购订单）
- ✅ API 网关（路由 + 鉴权 + 限流）
- ✅ 认证服务（JWT + OAuth2 + WebAuthn）

**不包含在内 (Out of Scope)**:
- ❌ 前端页面开发（仅提供 API）
- ❌ 移动端 APP
- ❌ 数据分析和 BI 报表
- ❌ 客服系统集成
- ❌ 财务结算模块

### 1.3 成功标准

| 维度 | 指标 | 目标值 | 权重 |
|-----|------|--------|------|
| **功能完整性** | 核心功能完成度 | 100% | 30% |
| **性能** | QPS/TPS 达标率 | 100% | 25% |

## 3. 项目进度计划

### 3.1 总体时间线（12周）

```
W1      W2-3          W4-5           W6-7           W8-9        W10       W11-12
│       │             │              │              │           │          │
├───────┼─────────────┼──────────────┼──────────────┼───────────┼──────────┤
Phase 0 │  Phase 1    │   Phase 2    │   Phase 3    │  Phase 4  │ Phase 5  │ Phase 6
基础设施│ 分布式事务  │ 商品+搜索    │ 库存高并发   │ 订单+状态机│仓库物流  │性能优化压测
```

### 3.2 详细里程碑

| 里程碑 | 日期 | 交付物 | 验收标准 | 责任人 |
|-------|------|--------|---------|--------|

| **M2: 商品搜索上线** | W5 结束 | 商品CRUD, ES搜索, Canal | 搜索QPS > 50000 | 后端组 |
| **M3: 库存服务上线** | W7 结束 | Redis Lua扣减, 分布式锁 | 扣减TPS > 10000 | 后端组 |
| **M4: 订单服务上线** | W9 结束 | 状态机, 创单流程 | 创单TPS > 10000 | 后端组 |
| **M5: 仓库物流上线** | W10 结束 | 入出库, 物流跟踪 | API响应 < 50ms | 后端组 |

## 6. 风险管理

### 6.1 Top 5 风险

1. **Seata 分布式事务性能瓶颈** (P1)
2. **Elasticsearch 集群稳定性问题** (P1)
3. **Redis 缓存穿透/雪崩** (P1)
4. **Nacos 配置中心故障** (P1)
5. **团队对新技术栈不熟悉** (P2)

详细风险评估见 [RISK_ASSESSMENT.md](./RISK_ASSESSMENT.md)

### 6.2 风险管理流程

1. **识别**: 每周风险评审会议识别新增风险
2. **评估**: 评估概率和影响，计算风险值
3. **应对**: 制定缓解策略和应急预案
4. **监控**: 跟踪风险状态变化
5. **关闭**: 风险解决或接受后关闭
